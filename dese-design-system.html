Foundations

Design Principles

Five principles shaped every decision in this system — from token naming to component behavior.

🎯
Information Density
Compact, data-heavy layout. No decorative whitespace. Every pixel earns its place in a workflow used for hours each day.
🔁
Ruthless Consistency
Same table header, same card border, same footer bar on every page. Predictability reduces cognitive load for daily users.
👤
Role Awareness
DESE sees all controls. Districts see pre-filled locked inputs. Public sees read-only dropdowns. One system, three contexts.
♿
Accessibility First
Every element has a visible focus ring. Color is never the sole affordance — links are always underlined. WCAG 2.1 AA throughout.
📐
Utilitarian Clarity
No illustrations, no gradients (except the dashboard hero), no decorative elements. This is a government tool used by finance professionals — the aesthetic is purposefully clean and institutional.
Foundations

Role Model

The UI adapts to three distinct roles. Components accept a userRole prop and render appropriately — locking inputs, hiding controls, or showing read-only states as needed.

DESE Administrator
Full system access. Can edit all data, add accounting notes, export all districts, manage submissions, and see system-wide views.
Full Access
District User
Scoped to own district. Can submit reports, view own data, and interact with permitted forms. Inputs pre-filled and locked where appropriate.
Scoped Access
Public / Guest
Read-only. Can view published district financial data. Dropdowns are filterable but non-editable. No submission capabilities.
Read Only

Accessibility

WCAG 2.1 AA compliance is non-negotiable for a government application. Every interactive element has a visible focus ring. Color is never the sole affordance.

Focus states — keyboard navigation demo ring-2 ring-blue-600
Input — focused state
Button — focused state
✅ Color + Underline for links
All links are both blue and underlined. Color is never the sole affordance — safe for color-blind users.
✅ Visible focus ring
ring-2 ring-blue-600 on every interactive element. Keyboard users always know where they are.
✅ Contrast ratio ≥ 4.5:1
All text/background combinations tested. Navy #2C4F7C on white passes at 8.2:1. Blue #2563EB on white passes at 5.1:1.
✅ ARIA labels & roles
All icon-only buttons have aria-label. Tables use proper scope attributes on headers.
