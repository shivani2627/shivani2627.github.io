The Problem

DAs managing criminal
cases in spreadsheets

Wisconsin District Attorneys across the state were tracking active criminal cases using a patchwork of spreadsheets, paper files, and disconnected tools. There was no centralised system, no shared visibility, and no way to efficiently manage the complex web of cases, hearings, defendants, and evidence that a modern DA's office handles daily.

I was brought in as the sole UI/UX designer on the PROTECT project to design their first dedicated case management platform — from brand identity to production-ready screens across both web and desktop applications.

Key Discovery

Desktop and web users had fundamentally different interaction patterns — DAs in courtrooms needed completely different flows than office staff managing case records. Solving that split drove the entire information architecture.

Discovery & Research

Understanding before
designing

Before designing a single screen, I ran workshops with District Attorneys, investigators, paralegals, and IT stakeholders to map the complete case lifecycle. This was not a typical B2B SaaS audience — understanding their mental models, legal constraints, and daily workflows required deep immersion.

🔍 Stakeholder workshops 🗺️ Case lifecycle mapping 👥 User interviews — DAs, investigators, staff 📋 Workflow documentation 🖥️ Desktop vs. web analysis
Design Scope

Five major efforts,
all solo

The PROTECT umbrella covered five distinct design workstreams, each with its own complexity and stakeholder requirements:

  • Brand Identity & Logo — Created the PROTECT brand from scratch, including logo, colour system, and design language that would work across both web and desktop contexts.
  • Desktop Application Revamp — Redesigned the core desktop case management app used daily by DA offices. Optimised for power users who navigate hundreds of cases.
  • Details Pane Web App — Designed the web-based details pane for case record management, balancing information density with clarity.
  • XCQ Web Application — Complex querying interface for cross-case search and filtering. Heavy information architecture work to make a powerful tool feel approachable.
  • eFiling Application — End-to-end eFiling portal for submitting court documents digitally, replacing a paper-based process with strict legal compliance requirements.

Key Learnings

What this project
taught me

  • Desktop UX is a different discipline. Desktop application design demands fundamentally different thinking than web — keyboard shortcuts, information density, window management, and multi-panel layouts require their own design patterns.
  • Domain expertise compounds. After 1.5 years embedded, my designs improved significantly because I understood the legal context as well as the users did.
  • Pattern libraries pay dividends. Building a reusable pattern library early meant the XCQ and eFiling modules came together faster and more consistently than the earlier workstreams.
